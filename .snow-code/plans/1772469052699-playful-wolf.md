# Plan: Upgrade Real-Time Communication to AMB

## Overview

Replace the current 3-second HTTP polling + heartbeat architecture with ServiceNow AMB (Asynchronous Messaging Bus). All session state changes will be pushed to clients instantly via AMB channels. The `is_online`/`last_seen` heartbeat will be retired in favour of AMB connection lifecycle events.

**Scope:** `x_902080_planningw` (all AMB channels declared in app scope)
**Presence:** AMB-only (no heartbeat fallback)
**Bots:** DemoSessionAjax actions publish AMB events

---

## Channel Topology

One channel per active session, published by the server on every state-mutating action.

| Channel | Topic | Published by | Payload summary |
|---------|-------|-------------|-----------------|
| `/x_902080_planningw/session/{sessionId}` | `session_state` | All mutating Ajax methods | Full `getVotingStatus` snapshot |
| `/x_902080_planningw/session/{sessionId}` | `participant_joined` | `SessionParticipantAjax.joinSession` | `{ userId, name, role }` |
| `/x_902080_planningw/session/{sessionId}` | `participant_left` | `SessionParticipantAjax.leaveSession` | `{ userId }` |
| `/x_902080_planningw/session/{sessionId}` | `presence_update` | AMB disconnect handler (server-side) | `{ userId, isOnline }` |

Using a single channel per session with a `topic` discriminator keeps subscription management simple on the client — one `addChannel` call per session.

## Files to Create

### 1. `src/server/script-includes/PlanningPokerAMB.js` — NEW
Server-side AMB publisher utility (Script Include, not client-callable).

```javascript
var PlanningPokerAMB = Class.create();
PlanningPokerAMB.prototype = {
    initialize: function() {},

    getChannel: function(sessionId) {
        return '/x_902080_planningw/session/' + sessionId;
    },

    publishSessionState: function(sessionId) {
        var ajax = new PlanningPokerSessionAjax();
        // Build the same lightweight payload as getVotingStatus but without GlideAjax wrapping
        var payload = ajax._buildVotingStatusPayload(sessionId, gs.getUserID());
        payload.topic = 'session_state';
        var amb = new GlideAMBManager();
        amb.publish(this.getChannel(sessionId), JSON.stringify(payload));
    },

    publishParticipantJoined: function(sessionId, userId, name, role) {
        var payload = JSON.stringify({
            topic: 'participant_joined',
            userId: userId, name: name, role: role
        });
        new GlideAMBManager().publish(this.getChannel(sessionId), payload);
    },

    publishParticipantLeft: function(sessionId, userId) {
        var payload = JSON.stringify({
            topic: 'participant_left', userId: userId
        });
        new GlideAMBManager().publish(this.getChannel(sessionId), payload);
    },

    publishPresenceUpdate: function(sessionId, userId, isOnline) {
        var payload = JSON.stringify({
            topic: 'presence_update', userId: userId, isOnline: isOnline
        });
        new GlideAMBManager().publish(this.getChannel(sessionId), payload);
    },

    type: 'PlanningPokerAMB'
};
```

```javascript
var ambClient = null;

function initAMB(sessionId) {
    var channelName = '/x_902080_planningw/session/' + sessionId;
    ambClient = new GlideAMBClient();
    ambClient.subscribe(channelName, function(message) {
        handleAMBMessage(JSON.parse(message.data));
    });
    ambClient.connect();
}

function handleAMBMessage(data) {
    var topic = data.topic;
    if (topic === 'session_state') {
        processVotingStatusUpdate(data);
    } else if (topic === 'participant_joined' || topic === 'participant_left') {
        loadSession();
    } else if (topic === 'presence_update') {
        updatePresenceIndicator(data.userId, data.isOnline);
    }
}

function processVotingStatusUpdate(d) {
    // Same logic as current pollVotingStatus callback (lines 2154-2228)
    // Structural change → loadSession(); otherwise → updateVotingIndicators()
    var structuralChange =
        (d.hasCurrentStory !== !!currentStoryId) ||
        (d.storyId !== currentStoryId) ||
        (d.storyStatus !== currentStoryStatus) ||
        (d.sessionStatus !== currentSessionStatus) ||
        (d.participants.length !== (sessionData && sessionData.participants ? sessionData.participants.length : 0));

    if (structuralChange) {
        loadSession();
    } else {
        updateVotingIndicators(d.participants);
        // propagate inline story edits if changed
    }
}

function handleAMBReconnect() {
    // On reconnect, fetch current state once via getVotingStatus (fallback poll)
    var ajax = new GlideAjax('PlanningPokerSessionAjax');
    ajax.addParam('sysparm_name', 'getVotingStatus');
    ajax.addParam('session_id', sessionId);
    ajax.getXMLAnswer(function(response) {
        var d = JSON.parse(response);
        processVotingStatusUpdate(d);
    });
}
```

## AMB Channel Registration (ServiceNow)

A new AMB channel record must be deployed via the Update Set. This is a `amb_channel` table record:

| Field | Value |
|-------|-------|
| Name | `/x_902080_planningw/session` |
| Application | `x_902080_planningw` |
| Supports Wildcards | `true` (allows `/session/{any-id}`) |
| Accessible by | Authenticated users (scoped to app roles) |

This record goes in `src/fluent/` or is deployed via a seeded data file. Implementation must create this record in the Update Set.
